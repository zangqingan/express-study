**版本**：1.0

## 1. 引言

### 1.1 项目背景

本项目为通用内容管理系统，基于现有MySQL数据库设计（含CMS核心、采集、RBAC、会员系统等模块），旨在提供一套完整、可扩展、高性能的内容管理解决方案。系统分为**前台门户**与**后台管理端**，支持多模板、自定义模型、付费会员及内容采集等高级功能。

### 1.2 产品目标

- 为运营人员提供灵活的内容发布与管理工具（文章、栏目、碎片、轮播等）。
- 实现基于RBAC的精细化权限控制，保障后台操作安全。
- 支持会员体系（注册、付费等级、订单、阅读记录），为内容变现提供基础。
- 具备内容采集能力（页面爬虫、API采集），辅助内容填充。
- 支持动态模型扩展（自定义字段），满足不同内容类型需求。

### 1.3 目标用户

| 用户角色           | 说明                                                       |
| ------------------ | ---------------------------------------------------------- |
| 游客               | 未登录的前台访客，可浏览文章、留言、注册                   |
| 普通会员           | 注册用户，可购买会员、阅读内容、查看阅读记录               |
| 付费会员           | 根据有效等级（月/季/年/永久），享有特权（如付费内容访问）  |
| 后台管理员（普通） | 管理部分内容（文章、栏目、留言等），权限受角色控制         |
| 后台超级管理员     | 拥有全部后台权限，可管理系统配置、用户、角色、菜单、模型等 |

### 2.2 核心业务流程示例

- **内容发布流程**：管理员登录 → 选择栏目 → 编辑文章（如绑定下载模型）→ 设置标签/属性 → 发布 → 前台可见。
- **会员付费流程**：会员注册 → 选择产品（等级）→ 生成订单 → 支付（支付宝/微信）→ 回调更新等级有效期 → 解锁特权内容。
- **内容采集流程**：配置采集规则（页面/API）→ 手动/定时执行 → 解析HTML/JSON → 存入指定栏目（草稿或发布状态）。

## 3. 前台功能需求

### 3.1 首页

- **轮播图**：展示 `cms_slide` 中启用的轮播图，支持点击跳转。
- **碎片区域**：调用 `cms_frag` 中的内容，如网站简介、广告位。
- **文章推荐**：聚合属性为“头条/推荐/热门”的文章，支持多标签切换。
- **最新文章**：按发布时间倒序展示。
- **友情链接**：展示 `cms_friend_link` 数据。
- **SEO信息**：读取 `cms_site` 中的标题、关键词、描述。

### 3.2 栏目/频道页

- 根据 `cms_category` 的 `type` 字段区分：
  - **栏目类型（type=0）**：展示该栏目下的文章列表（支持分页），列表模板可读 `list_view` 字段。
  - **单页类型（type=1）**：展示单篇文章内容（类似关于页），模板读取 `list_view` 或特殊标识。
- 支持子栏目导航（基于 `parent_id` 树形结构）。
- 若栏目绑定模型（`mid` 非空），列表展示对应扩展表数据（如下载模型显示文件名、版本等）。
- 支持按标签过滤（标签云或侧边栏）。

### 3.3 文章详情页

- 展示字段：标题、作者、来源、发布时间、阅读数（`pv`）、内容（`content`）。
- 关联栏目：显示当前栏目路径（面包屑）。
- 标签：展示关联标签（通过 `cms_article_tag`），支持点击标签查看相关文章。
- 上一篇/下一篇：基于同一栏目下的相邻文章。
- 阅读记录（会员专享）：登录会员阅读后，自动写入 `user_reading_record`；重复阅读不重复记录。
- 扩展模型内容：若文章所属栏目绑定模型（如下载模型），需额外展示 `extend_model_download` 中的字段（文件链接、版本等）。
- 模板支持：可单独指定 `article_view` 模板。

### 3.4 会员中心（需登录）

- **注册/登录**：支持用户名/邮箱/手机号注册，BCrypt加密；支持微信/QQ第三方登录（`user_social_login`）。
- **个人资料**：修改头像、昵称、密码等。
- **我的等级**：查看当前有效等级（从 `user_level_ship` 获取）及到期时间；展示等级权益。
- **订单管理**：列出 `user_order` 订单（待支付/已完成），支持重新支付或查看详情。
- **阅读历史**：展示 `user_reading_record` 与文章关联，可按时间倒序排列。
- **升级/续费**：展示 `user_product` 列表，选择产品下单（调用支付接口）。

### 3.5 留言板

- 表单提交（姓名、电话、标题、内容等），存入 `cms_message`。
- 前台展示审核通过的留言（可后续扩展）。

### 3.6 全局功能

- 站内搜索：按文章标题、简要内容模糊搜索，结果分页。
- 站点统计代码：在页面底部嵌入 `cms_site.code` 内容（如百度统计）。
- 多模板机制：根据 `cms_site.template` 及栏目/文章的模板字段动态加载视图。

## 4. 后台管理功能需求

后台采用 RBAC 权限模型，不同角色可见不同菜单。以下按模块列出：

### 4.1 仪表盘

- 数据概览：总文章数、今日新增、总会员数、总订单数、待处理留言数等。
- 快速入口：发布文章、查看最新订单等。

### 4.2 站点管理

| 子功能   | 对应表                                | 需求描述                                                                               |
| -------- | ------------------------------------- | -------------------------------------------------------------------------------------- |
| 站点设置 | `cms_site`                          | 编辑网站Logo、名称、域名、备案号、SEO信息、上传方式（普通/七牛云）等，仅单条记录维护。 |
| 配置字典 | `sys_config_type<br>``sys_config` | 管理配置分组（如微信小程序、七牛云），动态增删改配置键值对。支持状态控制。             |

### 4.3 内容管理

#### 4.3.1 栏目管理

- 树形展示无限级栏目。
- 字段：名称、拼音、路径、SEO信息、类型（栏目/页面）、排序、状态、关联模型（`mid`）、列表页/详情页模板。
- 操作：增删改查、排序、移动（修改 `parent_id`）。

#### 4.3.2 文章管理

- 列表展示文章，支持按栏目、状态、属性筛选。
- **编辑功能**：
  - 基础信息：标题、短标题、作者、来源、缩略图、简述。
  - 内容：富文本编辑器（支持HTML）。
  - 属性：头条/推荐/轮播/热门（多选，存储到 `attr` 位标记）。
  - 栏目：主栏目（`cid`），并可关联其他栏目（`sub_cid` 存储逗号分隔ID）。
  - 标签：选择已有标签或新增，同步更新 `cms_article_tag` 及 `cms_tag.ref_count`。
  - 扩展字段：若栏目绑定模型（如下载模型），动态展示模型字段表单（数据存至 `extend_model_download` 等扩展表）。
  - 自定义模板：可覆盖栏目级的 `article_view`。
- 操作：增删改查、复制、更改状态（发布/草稿）。

#### 4.3.3 标签管理

- 管理 `cms_tag`，查看标签引用次数，可合并标签（更新中间表及文章冗余 `tag_id`）。

#### 4.3.4 轮播管理

- 对 `cms_slide` 的增删改查，支持上传图片、设置链接和排序。

#### 4.3.5 碎片管理

- 管理 `cms_frag`：碎片名称、标识（代码调用key）、内容（富文本/文本框），支持在模板中直接调用。

#### 4.3.6 友情链接

- 管理 `cms_friend_link`：标题、链接、排序。

#### 4.3.7 留言管理

- 列表展示 `cms_message`，支持查看详情、回复（需新增回复表或字段，可后续扩展）、删除。

### 4.4 扩展模块

#### 4.4.1 模型管理（动态模型）

- **模型列表**：对应 `cms_model`，增删改查（注意删除模型时CASCADE删除 `cms_field` 记录，但需提示同时删除扩展表数据）。
- **字段管理**（一个模型对应多个字段，`cms_field`）：
  - 支持字段类型：单行文本、多行文本、下拉、单选、多选、日期、数字。
  - 设置字段长度、默认值、排序、选项配置（JSON存储于 `val` 列）。
  - **需要实现**：当字段增删改时，同步修改扩展表的结构（通过DDL语句：ADD COLUMN, DROP COLUMN, MODIFY COLUMN）。这是高级功能，需在数据库层面协调。

#### 4.4.2 采集管理

- **页面采集**（`plus_collect`）：
  - 配置目标URL、列表/标题/内容CSS选择器、分页规则、编码。
  - 支持自定义 `parse_data` JavaScript函数清洗采集结果。
  - 手动执行采集任务，将采集内容存入指定栏目（`cid`），可设置文章状态为草稿/发布。
- **接口采集**（`plus_gather`）：
  - 配置API地址、数据解析脚本（处理JSON返回，输出 `{title, content, ...}`）。
  - 支持手动执行。
- **安全要求**：解析脚本应在后端沙箱环境中执行，严格禁止危险函数；仅允许超级管理员配置采集任务。

### 4.5 会员管理

| 子功能   | 对应表                  | 需求描述                                                         |
| -------- | ----------------------- | ---------------------------------------------------------------- |
| 会员列表 | `user`                | 查看、编辑、禁用会员，搜索（按用户名、手机号）。                 |
| 等级管理 | `user_level`          | 定义等级代码（月/季/年/永久）、有效期天数、名称。                |
| 产品管理 | `user_product`        | 定义可购买的产品（价格、关联等级、有效天数、描述）。             |
| 订单管理 | `user_order`          | 查看订单列表，支持手动退款（更新状态为已退款）。可查看订单详情。 |
| 阅读记录 | `user_reading_record` | 查看会员阅读历史，用于运营分析。                                 |

### 4.6 系统管理（RBAC）

| 子功能   | 对应表            | 需求描述                                                                                |
| -------- | ----------------- | --------------------------------------------------------------------------------------- |
| 用户管理 | `sys_user`      | 管理后台登录账号，分配角色（`sys_user_role`）。                                       |
| 角色管理 | `sys_role`      | 定义角色（如超级管理员、普通运营），分配菜单/按钮权限（通过 `sys_role_menu`）。       |
| 菜单管理 | `sys_menu`      | 树形管理后台菜单及按钮权限（type=M目录/C菜单/F按钮），支持设置权限标识（permissions）。 |
| 登录日志 | `sys_login_log` | 记录后台用户登录IP、地理位置、时间。                                                    |
| 通知公告 | `sys_notice`    | 发布公告，前台可展示滚动通知（接口获取）。                                              |

### 4.7 其他功能

- **个人中心**：后台登录用户修改密码、头像等。
- **缓存管理**（可选）：手动清除站点配置、栏目树等缓存。

## 5. 非功能性需求

### 5.1 性能

- 前台首页加载时间 < 1.5秒（在优化缓存情况下）。
- 文章列表接口分页响应 < 500ms。
- 后台列表页（文章、订单）支持索引优化，避免全表扫描。

### 5.2 安全性

- 密码使用BCrypt加密存储（ `sys_user.password` / `user.password`）。
- 所有后台接口需基于JWT + RBAC权限校验，`sys_menu.permissions` 作为鉴权依据。
- 防SQL注入：使用参数化查询（如Knex）。
- 上传文件限制类型及大小，七牛云/本地存储均需防止恶意文件。
- 采集任务的 `parse_data` 脚本必须运行在隔离环境（如vm2），禁止文件系统访问。

### 5.3 可用性与兼容性

- 前台支持响应式设计（PC/手机端自适应）。
- 后台UI基于Vue3 + Naive UI，支持主流浏览器（Chrome, Firefox, Edge）。
- 多模板支持：通过 `template` 字段切换，模板文件放在约定目录。

### 5.4 可扩展性

- 动态模型引擎需保证扩展表的字段可自由增删，不影响已有数据。
- 插件化：关键业务（支付、采集）预留钩子，方便未来二次开发。

### 5.5 数据一致性

- 文章 `tag_id` 冗余字段与 `cms_article_tag` 中间表需双写，保证查询性能和关联完整性（由应用层事务维护）。
- 标签引用计数（`cms_tag.ref_count`）在关联增删时实时更新。

## 6. 数据字典及关键约束

详细数据字典见 `database_design.md`，以下为核心业务规则：

| 规则         | 说明                                                                                                        |
| ------------ | ----------------------------------------------------------------------------------------------------------- |
| 文章删除     | 若有关联标签（`cms_article_tag`）或扩展表数据，因外键为 `RESTRICT`，需先清理关联记录。                  |
| 栏目删除     | 若栏目下有文章（`cms_article.cid`）或采集任务依赖，外键 `RESTRICT` 阻止删除，需先转移或删除子内容。     |
| 模型删除     | `cms_model` 与 `cms_field` 为 `CASCADE`，删除模型将自动删除字段定义，但需应用层额外删除对应的扩展表。 |
| 会员等级过期 | 每日定时任务扫描 `user_level_ship.end_date`，将过期等级移动至历史或更新状态（可设计status字段）。         |
| 订单支付     | 支付成功后更新订单 `status`，并创建/更新 `user_level_ship` 记录（若不存在则新增，若存在则延长有效期）。 |
