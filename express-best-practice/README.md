# 二、Prisma集成

## 2.1 初始化

Prisma 是一个现代化的 ORM 工具，简化了数据库操作，并提供了强类型支持和自动生成的数据库模型。

Prisma 目前支持 PostgreSQL、MySQL、SQL Server、SQLite、MongoDB 和 CockroachDB。具体可查看官网。

在学习使用 Prisma ORM 将其连接到具体数据(如：MySQL) ，并生成 Prisma Client前，先学习几个核心概念。

* **Prisma Schema** ：项目的心脏，位于 `prisma/schema.prisma` 文件(自动生成)，以一个声明式的方式定义数据模型（Model也就是数据库中的表）、数据库连接和客户端生成器。
* **Prisma Client** ：一个**类型安全**的数据库客户端。它会根据你的 Schema 自动生成，提供完备的代码提示和类型检查，让数据库操作像普通函数调用一样安全。
* **Prisma Migrate** ：数据库迁移工具，负责将 `schema.prisma` 中模型的变更（如新增、修改表结构）同步到真实的数据库里。
* **Prisma Studio** ：一个方便的 GUI 工具，用来可视化管理数据库中的数据。

## 2.2 数据播种（seeding）

数据播种是指在数据库中插入初始数据的过程。在 Prisma 中，你可以使用 prisma db seed 命令来实现数据播种。
Prisma 的播种功能需要一个可执行的入口。从 Prisma ORM V7 开始，你必须显式地在prisma配置文件中为它配置一个命令。

```ts
// This file was generated by Prisma, and assumes you have installed the following:
// npm install --save-dev prisma dotenv
import "dotenv/config";
import { defineConfig } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations",
    seed: "tsx prisma/seed.ts"// 必须添加
  },
  datasource: {
    url: process.env["DATABASE_URL"],
  },
});

```

我们在prisma/lib/index.ts中实例化了一个 Prisma Client，并导出它。这样在其他地方就可以直接导入这个实例来操作数据库了。

```ts
// prisma/lib/index.ts
/** 
 * 实例化 PrismaClient 实例，并导出以供全局使用。
 * 这里我们使用了 PrismaMariaDb 适配器来连接 MySQL/MariaDB 数据库。
 * 如果你使用的是 PostgreSQL，可以安装 @prisma/adapter-postgres 并导入 PrismaPostgres 替换 PrismaMariaDb。
 */

import "dotenv/config";
import { PrismaMariaDb } from "@prisma/adapter-mariadb";
import { PrismaClient } from "../../generated/prisma/client.js";

const adapter = new PrismaMariaDb({
  host: process.env.DATABASE_HOST!,
  user: process.env.DATABASE_USER!,
  password: process.env.DATABASE_PASSWORD!,
  database: process.env.DATABASE_NAME!,
  connectionLimit: 5,
});
const prisma = new PrismaClient({ adapter });

export { prisma };

// prisma/seed.ts
// 引入prisma实例
import { prisma } from "./lib/index.js";

async function main() {
  console.log(`Start seeding ...`);

  // 使用 upsert 创建一个用户，如果邮箱已存在则跳过
  const alice = await prisma.user.upsert({
    where: { email: 'alice@prisma.io' },
    update: {},
    create: {
      email: 'alice@prisma.io',
      name: 'Alice',
      posts: {
        create: {
          title: 'Check out Prisma with Next.js',
          content: 'https://www.prisma.io/nextjs',
          published: true,
        },
      },
    },
  });
  console.log(`Created user with id: ${alice.id}`);

  // 批量创建用户示例
  const usersData = [
    { name: 'Bob', email: 'bob@prisma.io' },
    { name: 'Carol', email: 'carol@prisma.io' }
  ];
  const createdUsers = await prisma.user.createMany({
    data: usersData
  });
  console.log(`Batch created ${createdUsers.count} users`);

  console.log(`Seeding finished.`);
}

main()
  .then(async () => {
    await prisma.$disconnect();
  })
  .catch(async (e) => {
    console.error(e);
    await prisma.$disconnect();
    process.exit(1);
  });

```

配置好之后，在终端中执行以下命令即可为数据库填充数据。当然你也可以单独在package.json中添加scripts字段，然后执行npm run seed。

## 2.3 数据测试脚本

数据库连接成功，实例化客户端后就有一个客户端实例对象使用，可以通过这个对象来实现crud操作。定义一个测试脚本来验证数据库连接和基本的 CRUD 操作是否正常工作。

```ts
// prisma/query.ts
import { prisma } from "./lib";

async function main() {
  console.log('🔍 开始验证数据播种并演示 CRUD...\n');

  // ========== 1. 验证种子数据是否存在 ==========
  const userCount = await prisma.user.count();
  console.log(`📊 当前数据库中共有 ${userCount} 个用户`);

  if (userCount > 0) {
    const firstUser = await prisma.user.findFirst();
    console.log('👤 示例用户:', firstUser);
  } else {
    console.log('⚠️ 未发现种子数据，请先运行 `npx prisma db seed`');
  }

  // ========== 2. 创建 (Create) ==========
  const newUser = await prisma.user.create({
    data: {
      email: 'test@example.com',
      name: 'Test User',
    },
  });
  console.log('\n✨ 创建新用户:', newUser);

  // ========== 3. 读取 (Read) ==========
  const foundUser = await prisma.user.findUnique({
    where: { email: 'test@example.com' },
  });
  console.log('🔎 通过邮箱查找用户:', foundUser);

  // 查询所有用户（带条件排序）
  const allUsers = await prisma.user.findMany({
    orderBy: { id: 'desc' },
    take: 5,
  });
  console.log('📋 最近创建的5位用户:', allUsers);

  // ========== 4. 更新 (Update) ==========
  const updatedUser = await prisma.user.update({
    where: { email: 'test@example.com' },
    data: { name: 'Updated Name' },
  });
  console.log('✏️ 更新用户名:', updatedUser);

  // ========== 5. 删除 (Delete) ==========
  // 注意：通常生产环境不会直接删除种子数据，这里仅演示语法
  // 为避免破坏后续测试，我们删除刚刚创建的用户
  const deletedUser = await prisma.user.delete({
    where: { email: 'test@example.com' },
  });
  console.log('🗑️ 删除测试用户:', deletedUser);

  // 再次验证数量
  const finalCount = await prisma.user.count();
  console.log(`\n✅ 验证完成，最终用户数: ${finalCount}`);
}

main()
  .catch((e) => {
    console.error('❌ 错误:', e);
    process.exit(1);
  })
  .finally(async () => {
    await prisma.$disconnect();
  });

// 执行脚本 
$ npx tsx prisma/query.ts
```

# 三、模型定义相关

## 3.1 概述

一个model就是一个表(集合)，model中的每个字段就是表中的一个列。每个字段都有一个数据类型，Prisma支持多种数据类型，如字符串、整数、布尔值、日期等。你还可以定义关系字段来表示表之间的关系。
模型名单数大驼峰命名方式，通过 @@map("表名")，传入实际要创建的数据库表名。

定义如下：

```ts
model User {
  字段名    字段类型  可选其它约束条件
  id        Int      @id @default(autoincrement()) // 主键，自动递增
  email     String   @unique // 唯一索引
  name      String?  // 可选字段
  posts     Post[]   // 一对多关系，一个用户可以有多篇文章
  createdAt DateTime @default(now()) // 创建时间，默认值为当前时间

  @@map("sys_user") // 重命名-数据库实际表名。
}

// 在使用是通过小写的模型名实现crud操作，如：
prisma.user.xxx



```

**常见字段类型**

| 类型         | 描述           | 数据库默认映射 (例)                                     | 使用示例 / 备注                                                                                                                      |
| ------------ | -------------- | ------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `String`   | 文本字符串     | PostgreSQL: text`<br>`MySQL: varchar(191)             | `name String<br>``email String @unique`                                                                                          |
| `Int`      | 32位整数       | PostgreSQL: integer`<br>`MySQL: int                   | `age Int<br>``views Int @default(0)`                                                                                             |
| `BigInt`   | 有符号的大整数 | PostgreSQL: bigint`<br>`MySQL: bigint                 | 适用于可能超过32位整数范围的数值。在 Prisma Client 中，它会被表示为 JavaScript 的 BigInt 类型。                                      |
| `Float`    | 双精度浮点数   | PostgreSQL: double precision`<br>`MySQL: double       | `score Float`                                                                                                                      |
| `Decimal`  | 精确小数       | PostgreSQL: decimal(65,30)`<br>`MySQL: decimal(65,30) | 推荐用于需要高精度的金融数据等场景。在 Prisma Client 中，它由 Decimal.js 库提供支持。                                                |
| `Boolean`  | 布尔值         | PostgreSQL: boolean`<br>`MySQL: tinyint(1)            | `isPublished Boolean @default(false)`                                                                                              |
| `DateTime` | 日期和时间     | PostgreSQL: timestamp(3)`<br>`MySQL: datetime(3)      | 重要：在 Prisma Client 操作时，你必须传入 JavaScript 的 Date 对象，而不是字符串。常搭配 `@default(now())` 或 `@updatedAt` 使用。 |
| `Json`     | JSON 数据      | PostgreSQL: jsonb`<br>`MySQL: json                    | 适用于存储无固定结构的动态数据。在 Prisma Client 中，它被表示为任意的 JavaScript 对象/数组 (any)。                                   |
| `Bytes`    | 字节数据       | PostgreSQL: bytea`<br>`MySQL: blob                    | 用于存储二进制数据。在 Prisma Client 中，它被表示为 Uint8Array 类型                                                                  |



**其它可选属性**
| 属性                           | 作用                                                         | 示例                                     |
| ------------------------------ | ------------------------------------------------------------ | ---------------------------------------- |
| `@id`                        | 标记该字段为主键（自增需配合 `@default(autoincrement())`） | `id Int @id @default(autoincrement())` |
| `@unique`                    | 创建唯一约束，该字段值不能重复                               | `email String @unique`                 |
| `@default(value)`            | 设置默认值                                                   | `status String @default("active")`     |
| `@updatedAt`                 | 自动更新为当前时间戳（仅用于 DateTime 字段）                 | `updatedAt DateTime @updatedAt`        |
| `@map("column_name")`        | 映射自定义的数据库列名                                       | `firstName String @map("first_name")`  |
| `@@id([field1, field2])`     | 复合主键                                                     | `@@id([userId, roleId])`               |
| `@@unique([field1, field2])` | 复合唯一约束                                                 | `@@unique([firstName, lastName])`      |
| `@@index([field])`           | 创建普通索引                                                 | `@@index([email])`                     |
| `@@map("table_name")`        | 映射自定义数据库表名                                         | `@@map("users")`                       |

# 四、已存在数据库
连接已存在的 MySQL 数据库，Prisma 的处理方式有别于新建项目，核心区别在于：不直接使用 prisma migrate dev，而是通过 “内省” 来反向生成 Schema，并对已有数据库进行 “基线化” 处理。

```bash
# 安装 Prisma CLI、客户端及其所需的 MySQL 驱动适配器。
npm install prisma @prisma/client @prisma/adapter-mariadb --save-dev
# 初始化 Prisma
npx prisma init --datasource-provider mysql --output ../generated/prisma

# 环境变量文件 .env 中配置连接。
DATABASE_URL="mysql://USER:PASSWORD@HOST:PORT/DATABASE_NAME"

# 内省（Introspect）数据库：这是连接已有数据库最关键的一步。运行以下命令，Prisma 会读取你数据库的结构，并自动生成 Prisma Schema 数据模型
npx prisma db pull


Loaded Prisma config from prisma.config.ts.

Prisma schema loaded from prisma\schema.prisma.
Datasource "db": MySQL database "prisma-study" at "127.0.0.1:3306"

√ Introspected 2 models and wrote them into prisma\schema.prisma in 1.96s

Run prisma generate to generate Prisma Client.


# 内省完成之后，你的 Prisma schema 会包含代表现有数据库表的模型，接下来你可以生成 Prisma Client 用于数据库访问

# 为数据库设置基线（Baseline）：这是为了告诉 Prisma，当前的数据库结构是初始状态，以便未来用 Prisma Migrate 进行管理。你需要手动创建一个初始迁移，并将其标记为“已应用”
# 生成初始迁移脚本 
mkdir -p prisma/migrations/0_init
npx prisma migrate diff --from-empty --to-schema prisma/schema.prisma --script > prisma/migrations/0_init/migration.sql
# 这条命令会：从空数据模型（--from-empty）出发，与你的 prisma/schema.prisma 文件中的数据模型（--to-schema）进行比较，生成 SQL 脚本并输出到 prisma/migrations/0_init/migration.sql

# 标记为已应用：告诉 Prisma 这个迁移已经存在于你的数据库中了
npx prisma migrate resolve --applied 0_init

Prisma schema loaded from prisma\schema.prisma.
Datasource "db": MySQL database "prisma-study" at "127.0.0.1:3306"

Migration 0_init marked as applied.


# 生成 Prisma Client
npx prisma generate

```

# 五、常见cli命令
以下是 Prisma CLI 中常用的命令，涵盖开发、迁移、内省和数据库管理等多个环节。均以 npx prisma 形式运行。若已全局安装 Prisma CLI（npm install -g prisma），可直接使用 prisma <command>。不同版本可能有不同，使用 prisma --help 查阅当前版本的准确参数。



---

## 📦 项目初始化与环境配置

| 命令 | 说明 |
|------|------|
| `npx prisma init` | 在当前项目中初始化 Prisma，生成 `prisma/schema.prisma` 和 `.env` 文件 以及prisma.config.ts 文件 |
| `npx prisma init --datasource-provider <provider>` | 指定数据库类型（如 `postgresql`, `mysql`, `sqlite`）进行初始化 |
| `npx prisma generate` | 根据 `schema.prisma` 生成 Prisma Client（需在 schema 变更后执行） |

---

## 🔍 数据库内省（反向工程）

| 命令 | 说明 |
|------|------|
| `npx prisma db pull` | 连接已有数据库，内省（introspect）其结构并生成 Prisma 数据模型（生成 `schema.prisma`） |
| `npx prisma introspect` | （旧版本名称，现已整合到 `db pull`） |

---

## 🧬 数据库迁移（正向工程）

| 命令 | 说明 |
|------|------|
| `npx prisma migrate dev --name <migration_name>` | 开发环境创建并应用新迁移（基于模型变更自动生成 SQL） |
| `npx prisma migrate deploy` | 在生产/其他环境中应用所有待执行的迁移 |
| `npx prisma migrate resolve --applied <migration_name>` | 将指定迁移标记为“已应用”（用于对已有数据库进行基线化） |
| `npx prisma migrate resolve --rolled-back <migration_name>` | 将指定迁移标记为“已回滚” |
| `npx prisma migrate status` | 查看当前迁移状态（哪些已应用，哪些待应用） |
| `npx prisma migrate diff` | 计算两个数据源之间的差异，生成迁移 SQL（高级用法） |

---

## 🛠️ 数据库操作与工具

| 命令 | 说明 |
|------|------|
| `npx prisma db push` | 直接将 `schema.prisma` 中的模型推送到数据库（不生成迁移文件，适合原型开发） |
| `npx prisma db seed` | 执行种子脚本，向数据库填充初始数据（需在 `package.json` 中配置 `prisma.seed`） |
| `npx prisma db execute` | 对数据库执行原始 SQL 文件或字符串 |
| `npx prisma studio` | 启动 Prisma Studio（一个基于浏览器的图形化数据库管理工具） |

---

## 🧪 客户端生成与类型检查

| 命令 | 说明 |
|------|------|
| `npx prisma generate` | 生成 Prisma Client（每次修改 `schema.prisma` 后都应重新执行） |
| `npx prisma generate --no-engine` | 生成客户端但不下载 Query Engine（适用于某些无服务器环境） |
| `npx prisma validate` | 验证 `schema.prisma` 语法是否正确 |

---

## 📄 格式化与工具链

| 命令 | 说明 |
|------|------|
| `npx prisma format` | 自动格式化 `schema.prisma` 文件 |
| `npx prisma version` | 显示当前安装的 Prisma 各组件版本信息 |
| `npx prisma help` | 列出所有可用命令及帮助信息 |
